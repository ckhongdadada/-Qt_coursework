### 1.2 简历内容生成检查

`buildResumePreviewHtml()` 生成的分区（从上到下）：

```
1. 头部（姓名+摘要+联系方式+头像）     ← 手动填写
2. 求职意向（职位+方向+城市）           ← 手动填写，始终显示
3. 教育背景                            ← 手动 + 数据库课程(已完成)
4. 技能特长                            ← 手动填写
5. 项目经验                            ← 手动填写
6. 实践经历                            ← 手动 + 数据库经历
7. 竞赛获奖                            ← 手动 + 数据库成果
8. 角色任职                            ← 数据库角色(无手动字段)
9. 活动参与                            ← 数据库活动(无手动字段)
10. 补充内容                           ← 手动填写
```

**问题发现**：

#### ❌ 问题1：角色/活动分区没有手动编辑字段，但候选素材插入目标不明确

**代码位置**：`ResumeCandidatePanel.cpp` 第101-108行，`ResumePage.cpp` 第630-660行

**现象**：
- 候选面板选择"角色任职"时，插入按钮显示"插入到补充内容"
- 候选面板选择"活动参与"时，插入按钮也显示"插入到补充内容"
- `onInsertToSectionRequested()` 方法中没有 `section == "roles"` 和 `section == "activities"` 的处理分支

**原因**：
- 角色和活动分区完全来自数据库，没有对应的QTextEdit编辑字段
- 所以候选素材只能插入到"补充内容"区

**影响**：用户可能困惑，不知道角色和活动的候选素材该插入到哪里。

**建议修复**：为角色和活动添加手动编辑字段，或修改候选面板提示文字明确说明。

---

#### ⚠️ 问题2：教育背景中"已完成课程"列表可能过长

**代码位置**：`ResumePage.cpp` 第298-306行

**现象**：
```cpp
for (const Course& c : courses) {
    if (c.status == "Completed") {
        courseItems << QString("• %1（%2，%3 学分）").arg(...);
    }
}
```

**影响**：当前有32门已完成课程，全部列出会导致简历教育背景部分很长。

**建议修复**：只显示前10门分数最高的课程，或添加"查看更多"折叠功能。

---

#### ⚠️ 问题3：实践经历分区手动与数据库数据混合显示，无分隔

**代码位置**：`ResumePage.cpp` 第342-362行

**现象**：
- 手动填写的"实习经历"和数据库中的"经历"直接拼接在同一分区
- 中间只有`<br>`分隔，没有标题或视觉区分

**影响**：用户可能分不清哪些内容是手动填写的，哪些来自数据库。

**建议修复**：添加"手动填写"和"系统记录"两个小标题进行区分。

---

#### ⚠️ 问题4：活动分区默认不显示

**代码位置**：`ResumePage.cpp` 第232行，`ResumeEditorPanel.cpp` 第238行

**现象**：
```cpp
options["includeActivities"] = false;  // 默认false
```

**影响**：用户进入简历页时，活动分区默认隐藏，需要手动勾选才能显示。

**建议修复**：如果数据库中有活动数据，默认勾选显示。

## 二、各页面布局检查

### 2.1 总览页（OverviewPage）✅ 布局合理

- 标题 + 副标题 + 导出/清除按钮行
- 专业选择下拉框（条件显示）
- **8个指标卡片** 在4列网格中 → 合理
- **学分进度条** 在中间 → 合理
- 建议列表(2/3) + 学期走势(1/3) → 合理

**⚠️ 小问题**：专业下拉框在有数据时才显示，空数据时隐藏，可能导致页面高度变化。

---

### 2.2 课程库（CoursesPage）⚠️ 筛选器布局不对称

- 搜索框 + 状态筛选 + 类别筛选 + 排序 + 专业筛选 = 5个控件
- 放在 2行 x 3列 的网格中
- 第1行3个控件，第2行2个控件 → 右侧留空，视觉上不对称

**建议**：将5个筛选器排成一行（空间够的话），或调整网格布局为更平衡的形式。

---

### 2.3 角色职责（RolesPage）❌ 过滤器选项与数据库不匹配

**代码位置**：`RolesPage.cpp` 第42-49行

```cpp
m_typeFilter->addItem(zh("学生干部"), zh("学生干部"));
m_typeFilter->addItem(zh("社团负责人"), zh("社团负责人"));
```

**数据库实际值**：
```
Leadership, Academic, Technical
```

**影响**：用户选择"学生干部"过滤器时，无法匹配任何数据，因为数据库中的类型是"Leadership"。

**建议修复**：过滤器选项应与数据库实际值一致，或添加中英文映射。

---

### 2.4 目标管理（GoalsPage）⚠️ 排序输入框不够直观

- `m_sortInput` 是一个 `QLineEdit`，提示文字是"排序：progress / deadline / title / priority"
- 用户必须手动输入英文排序键，不够友好

**建议**：改为QComboBox下拉框，提供中文选项。

---

### 2.5 目标岗位（JobsPage）⚠️ 状态过滤器不够直观

- `m_statusInput` 是一个 `QLineEdit`，提示文字是"过滤激活状态"
- 用户必须输入"active"或"inactive"才能过滤

**建议**：改为QComboBox下拉框，提供"全部/关注中/已取消"选项。

---

### 2.6 分析报告（AnalysisPage）⚠️ 可能缺少滚动区域

- `SemesterAnalysisWidget` 最小高度420px
- `PeerBenchmarkWidget` 最小高度360px
- 合计最小高度780px，加上标题和指标区域约900px
- **页面没有放在QScrollArea中**

**影响**：如果窗口高度小于900px，底部内容会被截断，无法滚动查看。

**建议修复**：将整个内容放入QScrollArea中。

---

### 2.7 简历导出（ResumePage）✅ 布局合理

- QSplitter分隔左侧编辑面板(760px)和右侧候选面板(280px)
- 候选面板最小240px，最大320px → 合理
- 编辑面板在QScrollArea中 → 合理
- **潜在问题**：`setSizes({760, 280})` 总计1040px，如果窗口宽度小于1040px，Splitter会自动调整，但可能导致左侧编辑面板被压缩

---

### 2.8 数据导入（ImportsPage）✅ 布局合理

- 控制卡片（类别选择+文件选择+导入按钮）
- 预览卡片（表格+学分要求卡片）
- 已修复滚动位置保存问题

---

### 2.9 时间轴（TimelinePage）✅ 布局合理

## 三、问题汇总与修复优先级

### 🔴 高优先级（功能错误）

1. **RolesPage类型过滤器与数据库不匹配** → 过滤功能完全失效
2. **AnalysisPage缺少滚动区域** → 窗口高度不足时内容被截断

### 🟡 中优先级（UX问题）

3. **简历页角色/活动候选素材插入目标不明确** → 用户困惑
4. **GoalsPage排序输入框不直观** → 需要手动输入英文键
5. **JobsPage状态过滤器不直观** → 需要手动输入英文状态

### 🟢 低优先级（优化建议）

6. **简历教育背景课程列表过长** → 32门课程全部显示
7. **简历实践经历分区数据混合** → 手动与数据库数据无分隔
8. **简历活动分区默认隐藏** → 有数据时应默认显示
9. **CoursesPage筛选器布局不对称** → 视觉不美观
10. **OverviewPage专业下拉框显示/隐藏导致高度变化** → 页面跳动
