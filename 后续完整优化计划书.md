### 7.2 ImportService 结果结构标准化

[ImportService.h](C:/Users/28414/Desktop/cpp_project/src/service/ImportService.h) 目前主要靠 `QJsonObject` 返回结果。

从当前实现看，`ImportService` 还把这些事情塞在一个头文件里：

- CSV 文件逐行解析
- 表头别名归一化
- 各实体的字段映射
- 逐行落库和错误截断

这类逻辑已经明显超过“简单工具函数”的范围，适合单独拆出解析层和映射层。

建议新增结果模型：

- [src/model/ImportResult.h](C:/Users/28414/Desktop/cpp_project/src/model/ImportResult.h)

#### 建议字段

```cpp
struct ImportErrorItem {
    int row = 0;
    QString message;
};

struct ImportResult {
    int imported = 0;
    int failed = 0;
    QVector<ImportErrorItem> errors;
    QString message;
    bool success = false;
};
```

#### 作用

- 避免页面层反复 `result["xxx"]`
- 更利于后续单元测试
- 让 `ImportService` 从“头文件大函数”变成“解析 / 映射 / 结果对象”三段式结构
