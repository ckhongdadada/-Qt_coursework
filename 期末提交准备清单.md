# 软件体系结构与设计模式 - 期末提交准备清单

请在最终打包提交前，逐项核对以下材料和事项是否准备就绪：

## 一、 文件打包与提交要求

- [ ] **最终压缩包命名**：请确保将本项目文件夹下所有内容打包，压缩包命名必须严格为 **`【组长学号.zip】`**（例如：`2023123456.zip`）。
- [ ] **提交人与平台**：由 **组长** 统一提交至超星平台。

## 二、 压缩包内必须包含的材料

- [ ] **1. 期末大作业报告**：
  - 路径：`03_Reports/软件体系结构与设计模式-设计报告.md` (或同名的 `.docx` / `.pdf` 文件)。
  - **关键点**：封面格式必须符合老师附件要求；必须包含选择主题、体系结构风格、需求说明、用例分析、静态设计、详细设计、时序图、核心功能状态等。
- [ ] **2. 团队分工报告**：
  - 路径：`03_Reports/团队成员分工说明.md`。
  - **关键点**：需要明确反映针对详细设计部分的分工。
- [ ] **3. Git Commit 记录证明**：
  - **关键点**：必须提供代码托管平台（如 Github、Gitee）上的 Commit 记录截图或链接（若未放在报告中，建议单独建一个文档或图片存放）。
- [ ] **4. 完整的源代码**：
  - 路径：`01_SourceCode` 文件夹内。
- [ ] **5. 可直接运行的可执行文件**：
  - 路径：`02_Executable` 文件夹内。
  - **致命关键点**：可执行文件必须确保能够在**无需修改代码的情况下直接运行**并展示项目功能。如果可执行文件无法正常运行，**源代码部分不得分**。

## 三、 现场演示 (10分钟)

- [ ] **全员到场**：所有组员原则上均应到场。
- [ ] **答辩准备**：每位同学需要能够回答与本人负责部分相关的问题。
- [ ] **展示内容**：现场演示 + 口头讲解。
  - 核心包含：项目背景/分工、核心功能展示（至少3项）、**创新功能展示（至少1项）**、体系结构设计、质量意识体现（如输入校验/异常处理）、项目的可扩展性与可维护性。

## 四、 组间互评 (占期末大作业 30%)

- [ ] **每位成员独立提交**：这是一个针对个人的要求，**每个小组成员**都要独立在平台上提交对其他组的评分（满分10分）。
- [ ] **格式要求**：例如评价B组和C组，提交形式为 `B:8, C:6`。
- [ ] **致命关键点**：如果不提交该成绩，平台默认此项成绩为 **0分**！
